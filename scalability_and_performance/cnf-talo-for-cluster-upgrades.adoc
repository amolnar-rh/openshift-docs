= Topology Aware Lifecycle Operator for cluster updates
include::_attributes/common-attributes.adoc[]
:context: cnf-topology-aware-lifecycle-operator

toc::[]

You can use the Topology Aware Lifecycle Operator (TALO) to manage the software lifecycle of multiple single-node OpenShift clusters. TALO uses {rh-rhacm-first} policies to perform changes on the target clusters.
